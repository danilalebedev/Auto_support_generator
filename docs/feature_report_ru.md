# Краткий отчет по новым функциям Auto Support Generator

## Что уже добавлено

### GUI для обычного пользователя

Добавлен графический интерфейс, в котором пользователь выбирает:

- таблицу с соединениями: Word `.docx` с ChemDraw-объектами или CSV;
- zip-архив со спектрами;
- путь к `MestReNova.exe`, если автоопределение не сработало;
- выходной `.docx`.

Как пользоваться: запустить `SI Generator GUI.bat` или `AutoSupportGenerator.exe`, нажать `Load example` для проверки примера или выбрать свои файлы, затем нажать `Generate SI`.

### Масштабируемое окно GUI

Окно переработано так, чтобы помещаться на небольших экранах: основной контент теперь прокручивается, а нижние кнопки `Generate SI`, `Open output folder`, `Clear log` остаются видимыми. Диалоги выбора файлов открываются рядом с уже выбранными путями.

### Структурированный output

После генерации программа складывает результаты в понятные артефакты:

- `support_information.docx` - готовый Supporting Information;
- `processed_spectra.zip` - пакет обработанных спектров;
- `processed_mnova` - сохраненные `.mnova` файлы;
- `mnova_reports` - текстовые отчеты из Mnova;
- `logs` - предупреждения и служебные логи;
- `support_information.manifest.json` - manifest для проверки и дальнейшего patch workflow;
- `support_information.run_summary.json` - краткий отчет о запуске.

### Проверка экспериментальных данных

Опция `Check support (NMR, HRMS, elemental analysis)` включает проверку:

- соответствия числа H и C в NMR структурной формуле;
- совпадения рассчитанного и найденного HRMS;
- полноты и близости элементного анализа для элементов из формулы.

Если проверка находит проблему, она не останавливает генерацию, а добавляет предупреждение в `support_warnings.txt`, manifest/run summary и, где применимо, в текст SI.

### Настраиваемое оформление

Оформление переведено на единый Word-шаблон `SI_template.docx`.
Пользователь редактирует документ в Word почти в том же виде, в котором должен
получиться SI, а программа заменяет placeholder-ы на реальные данные.

Сейчас через шаблон настраиваются:

- поля страницы, шрифты, интервалы и отступы;
- жирный/курсив для заголовков, NMR, HRMS, IR и elemental analysis;
- подстрочные индексы в формулах, например `K₂CO₃`;
- надстрочные индексы изотопов;
- курсив для символа `J` в константах;
- текст reagent loadings без отдельного `Compound_characterization` template.


### Работа со спектрами Mnova

Для каждого соединения программа может обработать `1H` и `13C` спектры из zip-архива:

- построить изображения спектров;
- сохранить обработанные `.mnova`;
- добавить спектры в приложение SI;
- сохранить отчеты по извлеченному NMR-тексту.
- вставить спектры как PNG, как кликабельные PNG preview со ссылкой на `.mnova`
  или отключить spectra appendix.

Peak picking threshold теперь задается отдельно для `1H` и `13C` в GUI:
`1H threshold (%)` и `13C threshold (%)`. Значения по умолчанию: `6` для `1H`
и `4` для `13C`. Порог считается как доля от самого высокого не-solvent пика и
помогает не включать в peak picking слабые пики примесей/шума.

Если выбран режим `mnova`, в SI вставляется PNG preview спектра на том же месте,
что и в режиме `png`, но сама картинка становится гиперссылкой на обработанный
`.mnova` файл. Это дает пользователю нормальную визуальную страницу спектра и
возможность открыть обработанный Mnova-файл кликом по изображению.

Формат zip: папки с номерами соединений, например `2a`, `2b`, внутри которых лежат соответствующие спектры.

### Existing SI Tools

Добавлены инструменты для уже созданного SI:

- `Check` - проверить существующий manifest;
- `Renumber` - переименовать номера соединений, например `2a=3a,2b=3b`;
- `Remove` - удалить соединения из manifest/SI;
- `Reorder` - изменить порядок соединений.

Эти функции используют manifest, созданный при генерации.

### Preflight diagnostics

Перед долгим запуском программа проверяет типичные проблемы:

- отсутствует входной файл;
- неправильное расширение выходного файла;
- zip со спектрами не читается;
- spectra appendix включен, но zip не выбран;
- не найден MestReNova или служебный Mnova script.

Сообщения выводятся в `Run Log`, а для Mnova-ошибок показывается дополнительная строка `Details`.

### Внутренняя архитектура

Начат перевод старого линейного пайплайна на более модульную архитектуру:

- domain-модели и типы вынесены в `src/si_generator/domain`;
- graph nodes становятся тоньше и в основном оркестрируют шаги;
- request/config/profile/validation logic вынесены в отдельные доменные модули;
- старые импорты оставлены совместимыми, чтобы не ломать текущие сценарии.
